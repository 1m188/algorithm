# algorithm

这是一个算法题目集合，用于解决各种算法题目。

## 步骤
解决算法题目应该遵循的步骤：
- 根据提示词或者文件中的注释文档内容获取题目要求
- 设计算法解决相应的算法题目，并且编写代码
- 进行测试，要求覆盖主流用例和边缘用例
- 进行静态检查，要求不能够出现警告或者错误
- 编写详细的注释文档，包括整体思路和各个注意点，以增强可读性
- 对最终代码文件进行格式化以保持代码风格一致
- 最后在做完所有工作之后删除所有测试代码和编译产物等额外内容

## 约束
解题过程中应该遵循如下规则约束：
- 所有注释、文档、沟通、问答等内容均使用简体中文
- 不得修改或者删除文件中已有的注释等重要内容
- 写完代码之后应该添加详细的注释，包括整体解题思路和各个注意点
- 代码应该简洁直接，结构化、模块性好，不要添加额外冗余的内容或者结构（如不必要的import或者include等）。也不要为了潜在的扩展或者伸缩需求进行盲目的抽象或者分层。
- 代码不应该存在错误或者警告。
- 做完所有工作之后应该删除所有的测试代码和编译产物等额外内容，确保工作区干净
- 代码要求：
    - C/C++：C遵循C11标准，C++遵循C++17标准，格式化遵循 [.clang-format](./.clang-format) 文件
    - Python：版本请查看 [pyproject.toml](./pyproject.toml) 文件，格式化遵循black默认格式化要求
    - golang：版本请查看 [go.mod](./go.mod) 文件
    - rust：版本请查看 [Cargo.toml](./Cargo.toml) 文件
    - JavaScript/TypeScript：遵循ES6标准
    - 其他未提到的语言，其编码规范和格式化标准应该遵循各自在工业界内的最佳实践
